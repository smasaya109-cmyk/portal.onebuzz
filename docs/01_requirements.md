# 01. 要件定義

> ステータス: ドラフト(壁打ち中)。確定したら本文に「✅確定」を付ける。

## 1. 目的

`portal.onebuzz.net` で動く、複数のアプリを一覧で紹介し、各アプリ(`*.onebuzz.net`)へ送客するポータル(カタログ / LP)。
アプリの URL は順次追加していく運用のため、**追加・更新が容易な仕組み**を重視する。

### ★最優先の必須要件
アフィリエイト `aff_click_id` を親ドメイン Cookie(`.onebuzz.net`)に保存し、配下アプリへ引き継ぐこと。
これがこのサイトの機能上の核心であり、一覧表示等は付随要素。仕様は [06_affiliate_clickid.md](06_affiliate_clickid.md)。

## 2. ターゲットユーザー

- アプリを探している一般の閲覧者(ログイン不要)
- 日本語ユーザーを主軸に、英語ユーザーも対象

## 3. スコープ

### やること(In Scope)
- **★ aff_click_id の親ドメイン Cookie 引き継ぎ(middleware)** ← 最優先
- アプリ一覧の表示(カード形式を想定)
- 各アプリ(`*.onebuzz.net` サブドメイン)への送客リンク
- アプリ詳細ページ(個別紹介)
- カテゴリ/タグによる絞り込み・検索
- 日本語/英語/中国語の言語切り替え(ja / en / zh)
- レスポンシブ対応(スマホ/PC)
- SEO 基本対応(メタタグ・OGP・サイトマップ)

### やらないこと(Out of Scope)
- 閲覧者のユーザー認証・アカウント機能・ログイン
- 決済・課金・成果送信(`trackConversion`)← アプリ側の責務
- アフィリエイト SDK(`@smasaya109-cmyk/aff-sdk`)の導入 ← ポータルには不要
- `aff_click_id` を保存・引き継ぐ以外の用途で加工・送信すること
- アプリのレビュー・コメント投稿機能
- 自作の管理画面(当面は Supabase Studio で運用)

## 4. 主要機能(MVP)

| 機能 | 説明 | 優先度 |
|---|---|---|
| **aff_click_id 引き継ぎ** | `?aff_click_id` を親ドメイン Cookie に保存(middleware) | **★最優先** |
| アプリ一覧 | 公開中アプリをカード表示 | 必須 |
| アプリ詳細 | 名称・説明・画像・外部URL・カテゴリ | 必須 |
| カテゴリ絞り込み | カテゴリ単位で一覧フィルタ | 必須 |
| 多言語表示 | ja / en / zh の切替 | 必須 |
| カテゴリ/タグ絞り込み | カテゴリ + タグで一覧フィルタ | 必須 |
| 注目アプリ枠 | トップに is_featured のアプリを掲出 | 必須 |
| 検索 | 名称・説明のキーワード検索 | 必須 |

## 5. アプリ1件が持つ情報(項目案)

- アプリ名(ja / en / zh)
- 説明文(ja / en / zh)
- サムネイル/アイコン画像(**外部URL: `*.onebuzz.net`**)
- 外部リンク URL(アプリ本体 = サブドメイン)
- カテゴリ(1つ)
- タグ(複数)
- 公開状態(公開 / 下書き)
- 表示順 or 新着日時

→ 詳細なデータ構造は [05_database.md](05_database.md) で定義。

## 6. 非機能要件

- **パフォーマンス**: 一覧は静的生成(SSG/ISR)で高速表示
- **可用性**: Vercel のマネージド環境に依存
- **SEO**: 各アプリ詳細ページが個別 URL を持ち、インデックスされる
- **アクセシビリティ**: セマンティックHTML、キーボード操作、コントラスト確保
