# 01. 要件定義

> ステータス: ドラフト(壁打ち中)。確定したら本文に「✅確定」を付ける。

## 1. 目的

複数のアプリを一覧で紹介し、各アプリの紹介ページ/外部リンクへ誘導するポータルサイト。
アプリの URL は順次追加していく運用のため、**追加・更新が容易な仕組み**を最優先とする。

## 2. ターゲットユーザー

- アプリを探している一般の閲覧者(ログイン不要)
- 日本語ユーザーを主軸に、英語ユーザーも対象

## 3. スコープ

### やること(In Scope)
- アプリ一覧の表示(カード形式を想定)
- アプリ詳細ページ(個別紹介)
- カテゴリ/タグによる絞り込み・検索
- 日本語/英語の言語切り替え
- レスポンシブ対応(スマホ/PC)
- SEO 基本対応(メタタグ・OGP・サイトマップ)

### やらないこと(Out of Scope / 現時点)
- 閲覧者のユーザー認証・アカウント機能
- アプリのレビュー・コメント投稿機能
- 決済・課金
- 自作の管理画面(当面は Supabase Studio で運用)

## 4. 主要機能(MVP)

| 機能 | 説明 | 優先度 |
|---|---|---|
| アプリ一覧 | 公開中アプリをカード表示 | 必須 |
| アプリ詳細 | 名称・説明・画像・外部URL・カテゴリ | 必須 |
| カテゴリ絞り込み | カテゴリ単位で一覧フィルタ | 必須 |
| 多言語表示 | ja / en の切替 | 必須 |
| 検索 | 名称・説明のキーワード検索 | 任意 |
| 問い合わせ | フォーム → Resend で通知 | 任意(要相談) |

## 5. アプリ1件が持つ情報(項目案)

- アプリ名(ja / en)
- 説明文(ja / en)
- サムネイル/アイコン画像
- 外部リンク URL(アプリ本体)
- カテゴリ
- タグ(複数)
- 公開状態(公開 / 下書き)
- 表示順 or 新着日時

→ 詳細なデータ構造は [05_database.md](05_database.md) で定義。

## 6. 非機能要件

- **パフォーマンス**: 一覧は静的生成(SSG/ISR)で高速表示
- **可用性**: Vercel のマネージド環境に依存
- **SEO**: 各アプリ詳細ページが個別 URL を持ち、インデックスされる
- **アクセシビリティ**: セマンティックHTML、キーボード操作、コントラスト確保
